# 📋 프로젝트 요구사항 문서 (PRD)

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 개인 개발 블로그 |
| **설명** | Notion을 CMS로 활용한 개인 기술 블로그 플랫폼 |
| **목적** | Notion에서 글을 작성하면 자동으로 블로그에 반영되는 동적 블로그 시스템 구축 |
| **타겟 사용자** | 개인 개발자, 기술 블로거 |

---

## 🎯 핵심 가치 제안

### CMS로 Notion을 선택한 이유

- **낮은 진입장벽**: 개발자가 직접 DB를 관리할 필요 없음
- **편리한 UI**: 마크다운 기반의 친숙한 콘텐츠 작성 환경
- **자동 동기화**: Notion API를 통한 실시간 데이터 연동
- **확장성**: Notion 데이터베이스 구조 확장 용이
- **무료**: Notion API는 무료로 제공됨

---

## ✨ 주요 기능

### 1. Notion 데이터베이스 연동
- Notion API를 통한 안전한 데이터 연동
- 환경 변수로 API 키 관리
- 정기적인 데이터 동기화

### 2. 글 목록 페이지
- 발행된 모든 글의 목록 표시
- 최신 글부터 역순 정렬
- 글 미리보기 (제목, 카테고리, 발행일, 썸네일)
- 페이지네이션 또는 무한 스크롤

### 3. 글 상세 페이지
- Notion 페이지의 전체 콘텐츠 표시
- 마크다운 또는 HTML 렌더링
- 메타데이터 표시 (발행일, 카테고리, 태그)
- 이전/다음 글 네비게이션

### 4. 카테고리별 필터링
- 사이드바 또는 드롭다운을 통한 카테고리 선택
- 선택한 카테고리의 글만 표시
- 활성 카테고리 하이라이트

### 5. 검색 기능
- 제목 및 태그 기반 검색
- 실시간 검색 결과 표시
- 검색어 하이라이트

### 6. 반응형 디자인
- 모바일, 태블릿, 데스크톱 최적화
- 터치 친화적 UI
- 다양한 화면 크기 지원

## 📊 Notion 데이터베이스 구조

### 데이터베이스명
**Blog Posts** (또는 사용자 정의명)

### 필드 구성

| 필드명 | 데이터 타입 | 설명 | 필수 여부 |
|--------|-----------|------|---------|
| Title | Title | 글의 제목 | ✅ 필수 |
| Category | Select | 글의 카테고리 (React, Next.js, TypeScript 등) | ✅ 필수 |
| Tags | Multi-select | 글과 관련된 태그 (검색용) | ❌ 선택 |
| Published | Date | 글의 발행 날짜 | ✅ 필수 |
| Status | Select | 글의 상태 (Draft/Published) | ✅ 필수 |
| Content | Text | 글의 본문 (Notion 페이지 콘텐츠) | ✅ 필수 |
| Slug | Text | URL-friendly 글 식별자 (자동 생성 권장) | ✅ 필수 |

### Category 옵션 (예시)
- React
- Next.js
- TypeScript
- JavaScript
- Web Design
- DevOps
- 기타

### Status 옵션
- Draft (초안)
- Published (발행됨)

## 🎨 화면 구성

### 1. 홈 페이지 (`/`)
- **목적**: 최근 글 목록 표시
- **구성 요소**:
  - 헤더 (사이트 로고, 네비게이션)
  - 글 목록 (카드형)
    - 제목, 카테고리, 발행일, 썸네일
  - 사이드바 (카테고리 필터, 검색)
  - 푸터

### 2. 글 상세 페이지 (`/posts/[slug]`)
- **목적**: 선택한 글의 전체 내용 표시
- **구성 요소**:
  - 헤더
  - 글 제목 및 메타데이터
    - 발행일, 카테고리, 태그
  - 글 본문 (Notion 콘텐츠)
  - 이전/다음 글 네비게이션
  - 푸터

### 3. 카테고리 페이지 (`/category/[name]`)
- **목적**: 특정 카테고리의 글 목록 표시
- **구성 요소**:
  - 헤더
  - 카테고리 제목
  - 해당 카테고리의 글 목록
  - 푸터

### 4. 검색 결과 페이지 (`/search`)
- **목적**: 검색 결과 표시
- **구성 요소**:
  - 검색 입력창
  - 검색 결과 목록
  - "결과 없음" 안내 메시지
